Key Files

接手时优先看的文件

文件 职责 为什么重要
src/main.tsx 前端入口和根组件挂载。 从这里顺着路由和 provider 找主界面。
src/api/chat.ts 封装模型调用和消息发送。 AI 项目的真实行为通常集中在这里。
README.md 运行方式和项目意图。 和源码互相校验，防止只信文档。

Runtime Flow

把运行链路画出来

UI Event 用户点击、输入或上传。
State 组件状态或全局 store 更新。
API 请求后端、server action 或 worker。
AI / DB 模型、数据库、向量检索或外部服务。
Render 结果回到页面或文件输出。

每个箭头都必须能在源码里找到对应函数或文件。

Code Focus

只摘关键代码，不贴大段源码

src/lib/agent.ts 核心函数 
async function runAgent(input) {
  const context = await retrieveContext(input);
  const plan = await model.generate({ input, context });
  return executeTools(plan);
}
阅读结论

这一段决定系统行为

解释代码承担的职责、上游输入、下游副作用，以及改这里会影响哪些功能。

Risks / Next Steps

读完后要能指导下一步行动

A

风险点

未校验输入、硬编码配置、模型调用无重试、状态边界不清、缺测试。

B

先读路线

README -> package/config -> 入口 -> 路由 -> 核心 API -> 数据模型。

C

练手任务

加一个小功能、补错误态、整理配置、写 smoke test，优先选择低风险入口。

Takeaway

好笔记要让人能接手项目

不要把源码总结成一堆文件名。把事实组织成路径、关系、风险和下一步。
