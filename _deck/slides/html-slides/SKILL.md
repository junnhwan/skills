# HTML Slides — 源码 / 笔记 → 图文并茂的纸质风格 HTML deck

把一份代码项目（或文档、笔记）讲清楚：读进去 → 拆成若干页 → 用一套统一的纸质杂志风模板，**图文并茂**地呈现成一个**单文件 HTML**，双击即开、键盘翻页、可分享。

风格定位是纸质杂志感：米色纸感背景、墨色文字、霞鹜文楷字体、左侧自动生成的章节目录、固定画布 + 整体缩放（永远不变形）、元素逐级淡入。

## 何时用 / 何时别用

**用**：要把一个项目/主题做成「可翻页的可视化网页」讲给别人（或未来的自己）看。典型信号——「帮我读一下这个项目的源码，做个笔记」「做成 PPT 那样的 html」「图文并茂地讲一下这个项目」。

**别用**：只要一份纯文本文档/Markdown（没有翻页、没有可视化诉求）；要交互式 Web App（这只是一份静态展示稿）；要做真正可编辑的 .pptx（那是 Office）。

## 核心资产：从 skeleton.html 起步

`assets/skeleton.html` 是**起点**——它已经内置了完整的设计系统（全部 CSS）、导航 JS、字体加载，并包含 7 页**可直接预览的模板示例**。

**工作方式**：复制 `skeleton.html` 为目标文件 → 把示例 slide 替换成真实内容 → 完成。不要从零写 CSS/JS，那会丢失风格一致性。

> 在动手前，**先打开 `assets/skeleton.html` 看一眼**（或在浏览器里翻一遍示例页），建立对每种模板长什么样的直觉——这比读任何文字描述都管用。

## 工作流

### 1. 读素材，建立「可讲解的结构」

- 如果是源码：先摸清**模块组成**、**一次请求/数据的流转**、**关键设计决策**、**目录职责**、**值得提炼的哲学/注释**。这些天然对应不同模板。
- 如果是笔记/文档：提炼出**章节**、**并列要点**、**流程**、**对照表**、**金句**。
- 产出一张「这一页讲什么、用哪种模板」的草稿清单，再动 HTML。

### 2. 按映射规则选模板（决策表）

| 这一页要表达的内容 | 用模板 | 图文并茂的着力点 |
|---|---|---|
| 标题 + 一句引入 + 技术栈/关键词 | **Cover** 开场 | chips 标签 |
| 项目由 N 个模块组成 / 分层架构 | **module-map 网格** | 🟦 模块卡片网格 |
| 请求流程 / 数据流 / 调用链（有顺序） | **path-step 流程图** | → 横向带箭头 |
| 2–4 个并列要点 / 特性 / 决策 | **编号卡片 three-col** | 🎴 大号编号卡片 |
| 文件结构 / API 对照 / 资源清单 | **资料页 dense** | 📋 表格 + 竖链节点 |
| 设计哲学 / 关键注释 / 金句 + 代码佐证 | **引用页 + 代码块** | 💬 金句 + 💻 代码 |
| 时序 / ER / 类 / 复杂拓扑等系统设计图 | **mermaid-block** | 🕸 mermaid 渲染图 |
| 总结 / 下一步探索 | **Closing** | — |

**「图文并茂」是硬要求**：避免整页纯文字段落。每页尽量带一个视觉构件——流程图、模块网格、编号卡片、表格、节点链、代码块之一。读者翻到任何一页都该「有图可看」。

### 3. 基于 skeleton 填充

- 复制 `skeleton.html` → 目标文件。
- 替换 `<section class="slide">` 内容。结构永远是：`<section class="slide [修饰符]" data-title="页名" [data-nav="章节名"]>` → `<div class="slide-inner">` → 内容。
- 想要哪种模板，就**照搬 skeleton 里对应示例页的 HTML**，只换文字。所有 class 名都已定义好，照抄即可生效。
- 需要某个组件的精确写法 → 查 `references/components.md`；需要整页骨架 → 查 `references/templates.md`；要调配色/字体/尺寸 → 查 `references/design-system.md`。

### 4. 自检清单（交付前过一遍）

- [ ] 每页都有视觉构件，不是纯文字墙。
- [ ] 章节起点页加了 `data-nav="..."`（左侧目录靠它自动生成）。
- [ ] 新增的元素都带 `class="animate" style="--i: N"`，N 从 0 递增（逐级入场）。
- [ ] 文字不溢出画布（1720×900）；内容多就拆页或加 `dense` 修饰符，不要硬塞。
- [ ] 链接、代码、表格里的反引号/特殊字符已转义。
- [ ] 浏览器里翻一遍：键盘 ←/→ 能翻页、目录能跳转、字体加载正常。

## 必须遵守的约定（违反会直接破坏效果）

- **画布固定 1720×900**：内容用相对单位（`cqi`/`clamp`/`%`），不要写死 `px` 宽高，否则缩放后会错位。
- **目录靠 `data-nav`**：只有标了 `data-nav` 的 slide 才是章节起点；该章内其余 slide 的 `data-title` 自动成为目录子项。**不需要手写 index**——skeleton 的 JS 会自动生成目录。
- **`data-title` 必填**：每个 slide 都要有，目录和计数靠它。
- **一页一主题**：塞不下就拆成两页，比挤一页强。

## skeleton 已内置的能力（直接用，不用自己实现）

- **键盘**：←/→/Space 翻页、Home/End 首末页、1–9 跳页、`o` 概览网格、`f` 全屏。
- **目录每页可点**：章节首页是组头，子页是缩进子项，点击直达任意页。
- **代码自动高亮**：`.code-block` 的纯文本内容自动语法高亮（highlight.js 纸质主题）；`data-lang="go"` 指定语言。写法见 `references/components.md`。
- **章节分隔页**：`<section class="slide section-divider">`，Part 起点的大字过渡页。
- **系统设计图（mermaid）**：`.mermaid-block` 写 mermaid 源码，自动渲染时序 / ER / 类 / 流程 / 状态图（纸质主题）。写法见 `references/components.md`。
- **打印 / PDF**：浏览器 Ctrl/Cmd+P，每页一张，已隐藏侧栏与控件。
- **无障碍**：尊重系统「减少动效」偏好，自动禁用入场动画。

## 设计原则（决定观感）

- **纸质杂志感**：米色背景 + 墨色字 + 大量留白 + 衬线编号。别用纯白背景、别用饱和度高的色块。
- **克制用色**：强调色只用于卡片顶条、编号、关键词。正文统一墨色/灰。
- **逐级入场**：页面元素按 `--i` 顺序淡入，制造「逐步揭示」的讲述节奏——翻到一页，内容一个个出现，而不是一下子糊脸。
- **信息密度适中**：一页讲清一个点。`dense` 修饰符用于信息确实多的页（表格、对照），会收紧字号和间距。

## 深入查阅（按需加载，不必全读）

- `references/design-system.md` — 配色十六进制、字体栈、画布与缩放、动画延迟公式。**改主题色/字体时读。**
- `references/components.md` — 每个组件的完整 HTML 片段 + class + 修饰符 + 适用场景。**组装新页面时读。**
- `references/templates.md` — 7 种模板的整页骨架（可整段复制）+ 编排建议。**搭一整页时读。**

## 一句话总则

> 先打开 `assets/skeleton.html` 建立直觉 → 照搬对应模板的 HTML 换文字 → 保证每页有图、章节有 `data-nav`、元素有 `animate`。其余细节查 references。

## 进阶：面试笔记模式（与 project-grill-prep 协同）

当用户要的是**面试准备**（"帮我准备项目面试""面试拷打""简历项目问答"），不只是知识科普时：

1. **先做简历差距分析**：读简历对该项目的描述，逐条对照代码，标注 ✅ 有支撑 / ⚠️ 不够扎实 / ❌ 简历写了代码没有。
2. **每个技术亮点走五层**（背景痛点→选型→实现→Q&A话术→兜底），用 Q&A 卡片组件。详见 `references/templates.md` 的「面试笔记五层模板」；组件写法见 `references/components.md` 的「面试专用组件」。
3. **不写没压测证据的定量**（50ms / GB / 倍数）——面试红线，被追问测试方法会露馅。
4. **加简历对照页 + 简历红线页**（把面试官从简历的每个追问，都映射到笔记章节和代码证据）。

skeleton 已内置面试组件（Q&A 卡片 / 架构图 / 深度徽章 / 要点框）和放大字号。如果用户环境有 `project-grill-prep` skill，优先用它出内容（它有深度探测、面试官心理写法规范、简历差距分析），再用本 skill 套 HTML 呈现——两者互补。
