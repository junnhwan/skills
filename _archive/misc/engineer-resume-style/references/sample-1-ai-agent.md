# 样本 1：AI Agent 智能体（后端开发）

> 来源：用户提供的简历截图（D:\Download\1.png）
> 时间段：2025.08 - 2025.10

## 项目描述

本项目是一套面向业务应用系统级的综合智能解决方案，支持执行过程中的各项能力抽象并存储到数据库中。由配置和环境信息保存。

## 技术栈

Spring AI、SpringBoot、MyBatis、MySQL、PGVector、Redis、Docker

## 职责

- 构建 RAG 标签化知识库，支持上传文档、拉取 git 代码仓库，并使用 PGVector 编码文本和对应的向量
- 拆解 AI Agent 组件，定义出基于 Advisor、Tool、Prompt、Model、Api 等多设计相关元素，实现策略模式和线程池，并均衡向 AI Agent 相关功能化人格能力
- 设计 auto 对话分析模型，使 AI Agent 自动循环执行问题分析、自主规划、自动选择工具，并予以输出结果；通过执行人多轮对话有效处理输出，当结果运行不达预期时可重新进入
- 构建 MCP 服务模块，以 sse 方式向内置微信公众号 MCP、网站推文 MCP 等提供使用第三方 MCP 平台，搜索公司用工具如配置类型 MCP，配置自定义 MCP，落地为数据库中所有类型 MCP 服务的配置操作

## 这份样本好在哪

- **技术栈一行平铺**：用顿号或加号分隔，没有任何修饰词。读者扫一眼就知道用了什么。
- **每条职责动词开头**：构建 / 拆解 / 设计 / 构建 —— 朴素动词，不写"打造 / 锤炼 / 沉淀"这类修辞。
- **每个名词都有目的**：PGVector 出现是为了"编码文本和对应向量"，不是为了显得技术新；策略模式 / 线程池出现是为了"均衡 Agent 能力"，每个模式都跟一个具体动作绑定。
- **没有"对标"任何东西**：没有"对标 LangGraph"、"对标 Anthropic"。技术决策直接讲做了什么，让面试官自己去问"你是怎么想到用这个的"。
