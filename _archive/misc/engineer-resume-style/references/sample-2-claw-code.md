## 项目简介

一个企业级 AI 编程助手 CLI 工具，基于 Model Context Protocol (MCP) 协议，能够理解自然语言指令、自动调用工具、完成复杂编程任务的智能 Agent 系统。

## 技术栈

LangChain4j、Picocli、JLine 3、OkHttp、Jackson

## 技术亮点

1. **AI Agent 工程化落地**：实现完整 Agent 循环（感知→规划→执行→反馈），生产故障分析从 40 分钟降至 3 分钟（效率提升 92%）；通过结构化提示词与代码块解析，解决 DeepSeek 等大模型不原生支持 Function Calling 的痛点

2. **MCP 协议标准化实现**：以 C/S 架构 + JSON-RPC over stdio 方式落地 Model Context Protocol 标准，支持并行连接多个 MCP 服务器；构建标准化工具生态，支持 20+ 即插即用工具，工具集成成本从 N×M 降到 N+M

3. **智能交互体验**：Claude Code 式确认系统（智能 3 选项交互 → 创建 / 创建并运行 / 取消），直观展示 ai 操作工具，降低决策成本；Token-by-Token 实时流式输出，首字延迟 < 1 秒；内置直接命令执行器，可识别并直接执行高频系统命令，响应速度提升 10 倍；集成 JLine 3 提供命令补全、历史记录、状态栏显示等现代化 CLI 功能

4. **智能上下文管理**：系统提示词层 + 会话历史层 + 工具上下文层 + 项目上下文层四层架构；滑动窗口 + Token 控制 + 混合策略三种策略，智能控制上下文长度；支持会话完整生命周期管理：创建、加载、删除和自动保存

## 这份样本好在哪

- **每条亮点的硬数字都可以追问**：
  - "40 分钟降至 3 分钟" → 怎么测的、什么场景下、平均还是 p95？
  - "首字延迟 < 1 秒" → 第几秒、什么模型、网络条件？
  - "N×M 降到 N+M" → 这是协议化的标准成果，可以解释 MCP 为什么这样设计
  - "20+ 即插即用工具" → 哪 20+，哪些是自己写的哪些是 SDK 自带

  每个数字都为面试预留了一个"让我详细讲讲"的接口，没有一个是凭空修辞。

- **痛点写得具体**："解决 DeepSeek 等大模型不原生支持 Function Calling 的痛点" —— 把"为什么需要这个改造"讲清楚，比直接吹"实现 Function Calling"高级。
- **架构层级用 + 号串起来**："系统提示词层 + 会话历史层 + 工具上下文层 + 项目上下文层" —— 不写"四大维度"、"四层架构体系"，直接列层名。
- **同一条亮点里多个具体功能**：第 3 条用分号 `;` 连了 4 个子功能，每个都带数字或具体描述。这种"一条亮点 = 一个主题域 + 多个子功能"的结构能在简历有限空间里塞更多信息。
